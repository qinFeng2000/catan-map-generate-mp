# 地图 DOM 渲染、生成加载态与逐格展开设计

## 目标

页面地图改用 DOM 与 CSS 渲染，生成成功后让六边形地块从地图中心向外逐个展开。分享图片继续使用现有隐藏 Canvas，保持当前画面、尺寸和导出流程不变。所有需要寻找适配规格地图的入口统一使用 Taro 原生 Loading。

## 交互行为

- 首次进入且本地没有可恢复地图、切换 2–4 人 / 5–6 人规格、点击“重新生成”时，调用 `Taro.showLoading`。
- Loading 文案为“匹配规格地图”，开启 `mask`，避免搜索期间重复触发交互。
- 无论生成成功、返回无解还是抛出异常，都在 `finally` 中调用 `Taro.hideLoading`。
- 移除地图卡片内的“正在寻找满足规则的地图…”遮罩，避免两套加载反馈并存。
- 生成成功后，地图上的海洋与陆地六边形按各自中心到地图中心的距离排序，从中心向外依次缩放并渐显。
- 地块上的数字圆片和数字文字与所属地块同步出现；港口开口跟随所在海洋地块出现；图例不参与逐格动画。
- 切换人数规格与重新生成使用相同动画。
- 生成失败时保留旧地图，不重新播放动画。
- 隐藏的分享 Canvas 不参与动画。

## 渲染架构

### 共享场景命令

`createBoardScene` 继续作为页面与分享图共同的布局来源，生成包含颜色、坐标、尺寸和标签的 `DrawCommand`。因此 DOM 页面和 Canvas 分享图使用同一套投影结果，不分别维护地图几何与视觉参数。

### 页面 DOM 渲染器

新增 `BoardDom` 组件，接收页面已有的 `commands`、设计尺寸与渲染尺寸：

- `clear` 命令转换为地图根节点背景。
- `polygon` 命令转换为绝对定位的 `View`，使用包围盒定位和百分比 `clip-path: polygon(...)` 还原多边形。
- `circle` 命令转换为绝对定位、圆角为 50% 的 `View`。
- `text` 命令转换为绝对定位的 `Text`，沿用字号、字重、颜色和对齐方式。
- 坐标按场景宽高转换为百分比，根节点使用现有设计宽高，保证页面自适应缩放与 Canvas 布局一致。

页面只把可见的 `BoardCanvas` 替换成 `BoardDom`。离屏 `share-canvas`、Canvas 命令执行器和分享逻辑保持不变。

## 动画模型

DOM 渲染器根据六边形多边形的中心点计算其到场景中心的归一化距离，并转换为 CSS 自定义变量中的动画延迟。陆地、海洋地块采用相同的缩放与渐显关键帧；相同中心点的数字圆片、数字文字和港口开口复用该延迟。

页面使用成功地图的 `version`、`seed`、`createdAt` 组成 `key`。只有成功生成的新地图会让 `BoardDom` 重新挂载并播放动画；生成失败不会改变成功地图标识，因此旧地图保持静止可见。

动画使用较短的单格时长和受控的级联延迟，完整地图在约 600–800ms 内展开完成。动画只改变 `transform` 与 `opacity`，不改变最终坐标、尺寸或层级。

## 生成任务与错误处理

新增页面级生成任务包装器，负责 `showLoading`、调用生成器、异常归一化和 `hideLoading`。`useBoardGenerator` 继续负责状态切换与持久化，所有初次生成、规格切换和重新生成路径都经过该包装器。

生成器返回无解时沿用当前诊断信息；意外异常转换为“地图生成失败，请重试”。两种失败都不覆盖旧地图。

## 测试与验证

- 为生成任务先写失败测试，覆盖成功、无解和异常时 `showLoading` / `hideLoading` 配对行为。
- 为 DOM 命令转换先写失败测试，覆盖多边形、圆形、文字、坐标百分比和中心向外延迟。
- 验证数字与所属地块共享延迟，图例不参与逐格动画。
- 增加页面结构断言：可见地图使用 `BoardDom`，分享仍使用离屏 `BoardCanvas`，旧加载遮罩已移除，成功地图标识作为组件 `key`。
- 执行相关 Vitest、完整测试、TypeScript 类型检查、H5 构建和微信小程序构建。

## 验收标准

- 页面可见地图不再使用 Canvas，而由 DOM 与 CSS 完整渲染。
- 页面地图与当前 Canvas 分享图的地图比例、地块位置、颜色、数字和港口布局一致。
- 所有地图搜索入口显示 Taro 原生 Loading，并在所有结束路径关闭。
- 每次成功生成后，六边形从中心向外逐个展开；数字和港口跟随对应地块。
- 生成失败时旧地图保持可见且不重播动画。
- 分享图片继续由隐藏 Canvas 生成，输出设计保持不变。
